# Vllink WebUSB 调试器控制台 - 开发需求文档 (v1.1)

## 1. 项目概述
本项目是一个基于 WebUSB 协议的纯网页版硬件调试工具，专门用于管理 Vllink 系列调试器（Basic2 固件版本 >= V00.40）。工具实现了设备发现、有线/无线调试器状态监控、链路时长计算以及调试器节点切换功能。

## 2. 技术栈
*   **核心协议**：WebUSB API (`navigator.usb`)。
*   **前端框架**：原生 HTML5 / JavaScript (ES6+)。
*   **样式处理**：Tailwind CSS (CDN 引入)。
*   **设计风格**：极简科技感（参考 `dualsense-tester`），支持沉浸式磨砂玻璃效果（Glassmorphism）。

## 3. UI 布局规范
### 3.1 顶部状态栏 (Top Bar)
*   **左侧**：项目标题 `VLLINK CONSOLE`，带呼吸灯效果的状态指示点。
*   **中间**：三模态主题切换器。支持 **深色 (Dark)**、**自动 (Auto)**、**浅色 (Light)**。
    *   默认状态为 `Auto`（跟随操作系统主题）。
    *   UI 呈现为胶囊型滑块，切换时具备平滑的位移动画。
*   **右侧**：显示当前 WebUSB 连接状态（Disconnected / Online: 设备名）。

### 3.2 左侧面板 (宽度 25%)
*   **连接控制**：显眼的 `CONNECT DEVICE` 按钮，触发浏览器原生 WebUSB 选择框。
*   **设备列表**：动态展示当前所有在线节点（1个本地有线 + 最多9个远程无线）。
    *   **卡片设计**：包含别名（Alias）、连接类型（USB/WIFI）、运行时长（UPTIME）、响应延迟（Delay）。
    *   **高亮逻辑**：当前硬件选定的节点（`select_idx`）需呈现蓝色边框及外发光效果。

### 3.3 右侧功能区 (宽度 75%)
*   **选项卡导航**：
    1.  **配置**：当前为主功能页（内容待进一步指定）。
    2.  **待定**：预留功能占位。
*   **内容展示**：采用隐藏/显示切换机制，内容区具备磨砂玻璃质感。

## 4. WebUSB 通讯协议规范
### 4.1 接口识别
*   **VID/PID**：`{0x1209, 0x6666}` 或 `{0x0d28, 0x0204}`。
*   **接口过滤**：`interfaceClass == 0xFF` 且 `interfaceSubclass == 0x03`。
*   **独占模式**：必须执行 `claimInterface` 以确保通讯权限。

### 4.2 控制传输参数 (端点 0)
所有指令均通过控制传输完成：
*   **读取指令**：`bRequest == 0x00`, `bRequestType == 0x81` (Vendor, Interface, In)。
*   **写入指令**：`bRequest == 0x00`, `bRequestType == 0x41` (Vendor, Interface, Out)。
*   **数据包长度**：固定为 512 字节。

### 4.3 核心数据结构 (C 语言描述)
调试器查询返回的二进制数据结构（小端模式）：
```c
struct debugger_query_t {
    uint8_t select_idx;      // 当前选定的调试器索引 (0: local; 1-10: remote)
    uint8_t reserved[31];    // 头部填充

    struct node_t {
        uint64_t us;         // 时间戳 (微秒)
        uint32_t delay_us;   // DAP 指令延迟 (微秒)
        uint32_t reserved;   // 填充
        uint8_t alias[32];   // 别名 (字符串)
    } local, remote[9];      // 1个本地节点 + 9个远端节点
} __attribute__((packed));   // 总长度 512 字节
```

## 5. 关键业务逻辑
### 5.1 状态轮询
*   网页连接成功后，以 **250ms** 为周期循环发送读取指令（`bRequest: 0x00`）。
*   解析 512 字节 Buffer 时，需使用 `DataView` 处理 64 位无符号整数 (`BigInt`)。

### 5.2 运行时长 (UPTIME) 计算
*   **本地调试器 (ID 0)**：`时长 = debugger_info.local.us / 1000000`（秒）。
*   **无线调试器 (ID 1-9)**：`时长 = (debugger_info.local.us - debugger_info.remote[i].us) / 1000000`（秒）。
*   格式化输出：`HH:MM:SS`。

### 5.3 别名处理
*   对 `alias` 字段进行 `UTF-8` 解码并去除 `\0` 结束符。
*   若别名为空或仅包含空格，UI 需默认显示为 **"Unnamed"**。

### 5.4 节点切换
*   用户点击左侧列表卡片时，向设备发送写入指令。
*   `wValue` 的低字节设置为目标节点的 `index`，从而实现硬件层面的调试链路切换。
